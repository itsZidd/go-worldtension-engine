# WorldTension Data Engine

An automated ETL (Extract, Transform, Load) pipeline designed for geopolitical monitoring and data analysis. This engine ingests global news event streams, processes them using generative AI to derive quantitative stability metrics, and synchronizes the results with a centralized database.

---

## ⚙️ System Architecture

The engine operates as a scheduled serverless pipeline, executing the following lifecycle:

1. **Data Ingestion:** Downloads the latest 15-minute global event export from the **GDELT 2.0 Project** stream directly into memory.
2. **Code Mapping:** Translates legacy FIPS 10-4 country codes (used by GDELT) into ISO 3166-1 alpha-3 standards to ensure compatibility with modern mapping libraries and relational databases.
3. **AI Moderation:** Batches raw event data and utilizes **Gemini 3 Flash** to perform geopolitical sentiment analysis. The AI acts as a reasoning layer to transform qualitative headlines into structured data.
4. **Database Synchronization:** Executes a strict `UPSERT` operation on a **Supabase (PostgreSQL)** instance. This updates existing country records with fresh metrics while maintaining data integrity for legacy fields.
5. **Frontend Integration:** Sends a POST request to a **Vercel Deploy Hook**, triggering a static rebuild of the frontend intelligence library to reflect the new processed data.

---

## 📊 Data Methodology

The engine transforms qualitative news data into four primary geopolitical metrics:

- **Tension Score (-10 to +10):** Measures the immediate volatility of a region. Positive values indicate escalating conflict or diplomatic breakdown; negative values indicate stabilization or peace-building.
- **Stability (0-100%):** Assesses internal state health. High scores represent social cohesion and political order; low scores suggest civil unrest, protests, or institutional transition.
- **Industrial Capacity (0-100):** A composite metric reflecting economic output, infrastructure resilience, and resource mobilization potential.
- **Intelligence Briefing:** A natural language summary generated by the AI, synthesizing the day's most impactful events into a concise tactical report.

## 🚀 Getting Started

### Prerequisites

- Go 1.22 or higher installed locally.
- A Supabase project with a configured `world_tension` table.
- A Google AI Studio API Key.

### Local Environment Setup

Export the following environment variables in your terminal before running the engine:

```bash
# Database Connection
export DATABASE_URL="postgres://postgres.[ID]:[PASS]@aws-0-[REGION].pooler.supabase.com:5432/postgres"

# Google AI Studio API Key
export GEMINI_API_KEY="your-api-key-here"

# Vercel Rebuild Hook (Optional)
export VERCEL_DEPLOY_HOOK="your-webhook-url"
```

### Installation & Execution

1. **Clone the repository and install dependencies:**
   ```bash
   go mod tidy
   ```

2. **Run the pipeline manually:**
   ```bash
   go run .
   ```

## 🗄️ Database Schema Requirements

The engine targets a PostgreSQL table named `world_tension`. To ensure the "Future-Proof" data integrity logic works, the table must contain the following columns:

| Column | Type | Description |
| :--- | :--- | :--- |
| `iso_code` | TEXT (PK) | ISO 3166-1 alpha-3 code (e.g., USA, IDN) |
| `fips_code` | TEXT | Legacy FIPS 10-4 code (e.g., US, ID) |
| `tension_score` | NUMERIC | Current AI-calculated tension level |
| `stability` | INTEGER | Domestic stability percentage (0-100) |
| `industrial_capacity` | INTEGER | Economic mobilization metric (0-100) |
| `intel_report` | TEXT | AI-generated summary text |
| `event_count` | INTEGER | Total raw headlines processed for the turn |
| `last_updated` | TIMESTAMP | Automatic timestamp of the latest update |
